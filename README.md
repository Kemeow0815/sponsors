# 赞助页面

一个独立的赞助展示页面，支持明暗模式切换、响应式设计和灯箱查看二维码。

## 在线访问

本地测试：
```bash
python -m http.server 8080
```

然后访问 http://localhost:8080

## 如何提交赞助信息

如果您已经赞助，想将自己的信息添加到赞助者列表，请按照以下步骤操作：

### 1. Fork 本仓库

点击右上角的 "Fork" 按钮，将仓库复制到您的 GitHub 账户。

### 2. 创建赞助者数据文件

在 `data/sponsors/` 目录下创建一个新的 JSON 文件，文件名使用您的昵称（例如：`张三.json`）。

文件内容格式如下：

```json
{
  "name": "您的昵称",
  "date": "2025-01-15",
  "amount": "¥50",
  "avatar": "https://example.com/avatar.jpg"
}
```

字段说明：
- `name` (必填) - 您的昵称或姓名
- `date` (必填) - 赞助日期，格式为 `YYYY-MM-DD`
- `amount` (必填) - 赞助金额，例如：`¥50`、`$10`、`¥100.00`
- `avatar` (可选) - 头像图片 URL，支持 http/https 链接

### 3. 更新清单文件

编辑 `data/sponsors/sponsors.json`，将您的新文件名添加到 `files` 数组中：

```json
{
  "files": [
    "existing-sponsor.json",
    "张三.json"
  ]
}
```

### 4. 提交 Pull Request

1. 提交更改并推送到您的 Fork 仓库
2. 创建 Pull Request (PR) 到原仓库
3. 等待自动化检测和审核

## 自动化检测逻辑

本项目使用 GitHub Actions 进行自动化检测和处理。

### PR 自动检测 (`auto-pr.yml`)

当您提交 PR 时，系统会自动执行以下检测：

#### 1. 文件路径检查
- 只允许修改 `data/sponsors/` 目录下的文件
- 每个 PR 只能包含一个文件变更

#### 2. 文件格式检查
- 文件必须是 `.json` 格式
- JSON 必须能被正确解析
- JSON 必须是一个对象（不是数组或其他类型）

#### 3. 字段校验
- `name` 必须是字符串且不能为空
- `date` 必须是字符串且不能为空
- `amount` 必须是字符串且不能为空
- `avatar` 如果存在，必须是有效的 URL 字符串

#### 4. 头像链接检测
- 如果提供了 `avatar` 字段，系统会尝试访问该链接
- 头像必须可访问（HTTP 200 状态码）
- 不可访问的头像会导致 PR 被标记为 "头像不可达"

#### 5. 自动合并
- 如果所有检查通过，PR 会被自动合并
- 合并后您的信息会立即显示在赞助页面上

### 定期链接检查 (`check-links.yml`)

系统会每 30 天自动运行一次链接检查：

1. 遍历所有赞助者数据文件
2. 检查每个 `avatar` 链接的可访问性
3. 如果头像链接不可访问，自动删除该赞助者条目
4. 更新 `sponsors.json` 清单文件

**注意**：如果您的头像链接失效，您的赞助信息将被自动移除。请确保使用稳定的头像托管服务。

### SSRF 保护

为了防止服务器端请求伪造攻击，系统会检查所有 URL：
- 只允许 `http://` 和 `https://` 协议
- 禁止访问本地地址（localhost、127.0.0.1）
- 禁止访问私有 IP 段（10.x.x.x、192.168.x.x、172.16-31.x.x）

## 目录结构

```
sponsor/
├── index.html              # 主页面
├── README.md               # 本文件
├── data/
│   └── sponsors/
│       ├── sponsors.json   # 赞助者清单
│       └── *.json          # 赞助者数据文件
├── images/
│   ├── alipay.avif        # 支付宝二维码
│   └── weixin.avif        # 微信支付二维码
├── js/
│   └── lightbox.js        # 灯箱插件
└── .github/
    └── workflows/
        ├── auto-pr.yml     # PR 自动处理
        └── check-links.yml # 链接检查
```

## 技术特性

- **纯静态页面** - 无需后端服务器，可直接部署到任何静态托管服务
- **响应式设计** - 适配桌面端和移动端
- **明暗模式** - 支持深色/浅色主题切换，偏好自动保存
- **灯箱查看** - 点击二维码可放大查看
- **自动化流程** - PR 自动检测和合并，定期链接检查
